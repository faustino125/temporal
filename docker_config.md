# Docker Image Pull and Configuration for DataEngineerHub

This document outlines the steps to pull the Docker image for Prefect, set up the development environment in VS Code, and run the Prefect workflows in Databricks.

## Prerequisites

- Access to Azure CLI
- Docker installed and running
- Visual Studio Code (VS Code) installed
- Docker Desktop installed
- Extensions: "Dev Containers" in VS Code
- Proper access to the Azure DevOps repository and Databricks

## 1. Login to Azure CLI

To pull the Docker image from Azure, first log in using the Azure CLI.

Run the following command and select the "Banco Industrial Prod" subscription:
```bash
az login
```

Then, log in to the container registry by running:
```bash
az login --name imagenesanalitica
```

> **Note:** For this command, ensure you are **outside the bank's network** and that the **proxy is disabled** in your network settings.

## 2. Pull the Updated Docker Image

Once logged in, pull the latest Docker image that contains the Prefect setup. Run the following command:
```bash
docker pull imagenesanalitica.azurecr.io/bi/dev-prefect:latest
```

## 3. Run the Docker Image

After pulling the image, open Docker Desktop, search for the `DEV-PREFECT` image, and start a container by selecting "Run" for that image.

## 4. Set Up VS Code for Development

### Install the "Dev Containers" Extension

If you do not already have it, install the "Dev Containers" extension in VS Code. You can do this by searching for "Dev Containers" in the VS Code Extensions marketplace.

## 5. Connect VS Code to the Running Container

Ensure your `DEV-PREFECT` container is running, then follow these steps:

1. In the lower-left corner of VS Code, click on the button labeled **"Open a Remote Window"**.
2. Select **"Attach to Running Container"**.
3. From the list, choose the container corresponding to `DEV-PREFECT`.

## 6. Clone the Project Repository

Once connected to the running container, open a terminal within VS Code and create a directory named `repo`:
```bash
mkdir repo
```

Navigate into the `repo` directory and clone your repository using the following command:
```bash
git clone https://Bi-McKensey@dev.azure.com/Bi-McKensey/Data%20Factory/_git/DataEngineerHub
```

When prompted, enter your Azure DevOps token. This token can be generated by going to **Azure DevOps** → **Project Data Factory** → **DataEngineerHub Repository** → **Clone** button → **Generate a new token**.

## 7. Open the Project in VS Code

Navigate to the newly created `DataEngineerHub` folder:
```bash
cd DataEngineerHub
```

Then, open the project in VS Code:
```bash
code .
```

## 8. Install the "Databricks" Extension

Within VS Code, install the **Databricks** extension from the marketplace. Once installed, follow these steps:

1. Open the **Databricks** extension pane in VS Code.
2. Click on **"Create a new Databricks Project"**.
3. Select the appropriate host for your container.
4. Set up a new authentication profile using a **Personal Access Token**. You can generate this token in **Databricks** by going to **Settings** → **Developer** → **Access Tokens** → **Generate a new token**.
5. Name the profile as desired.

Ensure the project is initiated from the base folder of the repository `DataEngineerHub`.

## 9. Set Up Databricks Project

Once the authentication is complete, you'll be prompted to choose a template for the project. Select the template named **"default-python"** and name the project `databricks_bundle`.

When asked for additional setup options, choose **No** for the remaining setup options.

After setup, close the terminal window and select the `databricks_bundle` project. Your Databricks environment should now be properly configured.

## 10. Deploy a Workflow Using Prefect

Once the environment is set up and the Prefect framework from the `DataEngineerHub` repository is ready, you can deploy a workflow to Databricks.

Run the following command to deploy:
```bash
databricks bundle deploy --profile <AUTHENTICATION_PROFILE>
```

Replace `<AUTHENTICATION_PROFILE>` with the name of the Databricks authentication profile you set up.

## 11. Run a Deployed Workflow

To run the deployed workflow, execute the following command:
```bash
databricks bundle run --profile <AUTHENTICATION_PROFILE>
```

After running this command, select the workflow you just created and it will begin executing in Databricks.

---

Following these steps will ensure that you have the Docker container running, the development environment configured in VS Code, and Prefect workflows successfully deployed and running in Databricks.
